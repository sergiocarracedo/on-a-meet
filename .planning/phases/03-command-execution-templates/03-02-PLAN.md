# Plan 03-02: Wire Executor into Detect Command

<objective>
Connect the executor to the engine's OnChange callback so that real camera state transitions trigger command execution with template substitution. Add `--timeout` flag to the detect command. Update config.yaml.example. Commands run in goroutines to avoid blocking the poll loop. The final result is demoable: `./on-a-meet detect --on 'echo cam-{{.CameraID}}-{{.State}}'` prints the rendered command output when the camera toggles.
</objective>

3. Import `internal/executor` and `context`.

4. In the `RunE` function, between the camera banner output and the engine creation, parse the timeout and create the executor. Then wire it in the OnChange callback using goroutines:

```go
// Parse timeout
timeout, err := time.ParseDuration(cfg.Timeout)
if err != nil {
    return err
}

// Create executor (shared across all camera state changes)
exec := executor.New(timeout)

// Create engine with OnChange callback that fires commands
eng := engine.New(det,
    engine.WithInterval(interval),
    engine.WithDebounce(cfg.Debounce),
    engine.WithOnChange(func(path string, oldState, newState bool, info detector.DeviceInfo) {
        // Hotplug events (oldState == newState) are ignored — no commands fire
        if oldState == newState {
            return
        }

        data := executor.TemplateData{
            CameraID: path[5:], // strip "/dev/" → "video0"
            Device:   path,
        }

        if newState {
            data.State = "on"
            if cfg.OnCmd != "" {
                go func() {
                    if err := exec.ExecOn(context.Background(), cfg.OnCmd, data); err != nil {
                        output.Warning.Printfln("on-command failed: %v", err)
                    }
                }()
            }
        } else {
            data.State = "off"
            if cfg.OffCmd != "" {
                go func() {
                    if err := exec.ExecOff(context.Background(), cfg.OffCmd, data); err != nil {
                        output.Warning.Printfln("off-command failed: %v", err)
                    }
                }()
            }
        }
    }),
)
```

Key design decisions:
- Commands run in goroutines so they never block the poll loop
- Hotplug events (oldState==newState) are ignored — no commands fire on device add/remove, only on state transitions
- Empty on-command or off-command strings are silently skipped
- Command errors are reported via pterm Warning but don't stop polling
- The executor is given `context.Background()` so that in-flight commands aren't killed on engine shutdown. The executor's own per-command timeout handles runaway commands.
