# Plan 06-02: Sudo Apply + Service Install

## Tasks

### Task 1: Extract shared service helper from install.go

**Files:**
- `cmd/install.go`

**Action:**
Extract the service installation logic from `installCmd.RunE` into a shared function `installService() error` that can be called from both `install` and `onboard` commands. The function should:
1. Create the kardianos/service
2. Call `svc.Install()`
3. Call `svc.Start()`
4. Return any error

Keep `installCmd.RunE` as a thin wrapper that calls `installService()`.

**Verify:**
- `go build ./...` succeeds
- `go vet ./...` passes

**Done:**
- `installService()` function exists and is callable from `cmd/onboard.go`

### Task 2: Add --apply flag and sudo execution path

**Files:**
- `cmd/onboard.go`

**Action:**
Add `--apply <file>` flag to `onboard` command. When set:
1. Read the JSON file from `<file>` path
2. Unmarshal into a struct matching the collected values (cameras, method, debounce, interval)
3. Construct a `detectConfig` (or similar) struct
4. Ensure `/etc/on-a-meet/` directory exists (`os.MkdirAll`)
5. Marshal config to YAML using `gopkg.in/yaml.v3` (add to go.mod if needed)
6. Write to `/etc/on-a-meet/config.yaml`
7. Call `installService()` to install and start the service
8. Print `output.Success.Printfln("Config written to /etc/on-a-meet/config.yaml")`

In the non-sudo flow (from Plan 06-01), after writing collected answers to `/tmp/on-a-meet-onboard.json`, instead of printing instructions:
1. Find the binary path via `os.Executable()`
2. Print `output.Info.Printfln("Running with elevated privileges to complete setup...")`
3. Run `exec.Command("sudo", binary, "onboard", "--apply", "/tmp/on-a-meet-onboard.json")`
4. Connect stdout/stderr passthrough for the user
5. Wait for completion

**Verify:**
- `go build ./...` succeeds
- `go vet ./...` passes
- `sudo ./on-a-meet onboard --apply /tmp/on-a-meet-onboard.json` writes config and installs service

**Done:**
- `--apply` flag functional for sudo path
- End-to-end: user runs `on-a-meet onboard` → wizard → sudo re-exec → config written → service installed → path printed
