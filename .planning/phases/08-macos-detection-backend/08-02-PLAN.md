## Interactive Setup

on-a-meet includes an interactive setup wizard:

```bash
on-a-meet onboard              # Full interactive wizard
on-a-meet onboard --dry-run    # Preview config without applying
```

The wizard guides you through:
1. Selecting cameras to monitor (multi-select)
2. Choosing detection method (v4l2, lsof, or darwin on macOS)
3. Live test of the selected detection method
4. Configuring on/off commands
5. Preview and apply configuration
6. Optional service installation

On Linux, the sudo apply path writes config to `/etc/on-a-meet.yaml`
and installs the system service automatically.

```markdown
### macOS

Download the darwin binary from the [releases page](https://github.com/anomalyco/on-a-meet/releases).

```bash
# Download and make executable
curl -L -o on-a-meet https://github.com/anomalyco/on-a-meet/releases/latest/download/on-a-meet_darwin_amd64
chmod +x on-a-meet
sudo mv on-a-meet /usr/local/bin/
```

#### Permissions

macOS detection uses the built-in `log` and `system_profiler` commands — no
additional permissions required for detection. Camera access is read via
system logs, not direct hardware access.

The `on-a-meet` binary does not need camera permission (no
`NSCameraUsageDescription`) since it reads system logs rather than
accessing the camera hardware directly.
```
